eclipse 获取并运行git上项目

By you

2021-04-12 00:00:00 +0000 UTC

背景介绍

公司运用GitLab来管理Java的源码，在管理员授权源码权限后，可以获取最新的程序到本地运行。记录下来配置的流程，方便以后查看。      本地操作系统是win10，安装好Git的环境，安装比较简单，可以参考网上的安装教程，这里不再列举，推荐学习下廖雪峰老师的教程，链接是参考博客【1】。

第一步、ssh 生成本地 key

查看本地key是否生成成功，文件路径在 C:\Users\.ssh\id_rsa.pub 若没有，则执行以下命令

ssh-keygen -t rsa -C "你的邮箱"

将 id_rsa.pub 文件中的内容粘贴到账户的SSH中，关联你的账户。同时，记得设置用户名和邮箱，在commit 提交时会带上个人的信息。

// 设置用户名
$ git config --global user.name"你的用户名"
// 设置邮箱
$ git config --global user.email"你的邮箱"

本地生成的id_rsa.pub文件内容如下图所示。

第二步、导入项目

以eclipse开发工具为例，下面介绍如何获取服务器上最新的项目代码。新建工作空间，复制项目的git地址。右键 –> import 导入，选择项目导入方式为Git，如下图所示。 选择项目导入方式为Git 自动识别git的信息 直接下一步，选择存放项目的本地位置，就可以下载服务器的上源码文件。

注意：在程序开发前，要及时获取服务器上最新代码，一旦同时操作相同文件就可以造成冲突，处理冲突问题也是程序员必备的技能。

右键 –> Team –> pull 获取服务器上最新程序 解决冲突的方式是对比本地和服务器上代码的差异，右击项目 -> Team -> Sychronized WorkSpace，双击文件会展示两者的异同。

处理冲突的方式可参考博客【2】
