初探 Prometheus安装配置篇

By you

2021-10-02 00:00:00 +0000 UTC

背景介绍

周末在家搭建普罗米修斯监控，记录下来，以备查看。

知识学习 Prometheus 作为一个开源的服务监控系统和时间序列数据库，开源真好。 特点： 1）多维度数据模型：由度量名称和键值对标识时间序列数据 2）promSQL 数据查询 3）不依赖分布式环境，单个服务器节点可以直接工作 4）基于HTTP的pull方式才是时间序列数据 5）推送时间序列数据通过PushGateWay组件 6）通过服务发现或静态配置发现目标 7）多图模式及仪表盘支持

Prometheus 安装部署

安装方式有两种：二进制安装和docker安装 1、二进制安装 到官网下载二进制文件，下载地址 Prometheus 官网，将下载的gz文件放到服务器上。

// 解压
tar -zxvf prometheus-2.28.1.linux-amd64.tar.gz
// 新建文件
mkdir /opt/prometheus
// 移动
mv prometheus-2.28.1.linux-amd64/* /opt/prometheus
// 将默认端口9090添加到信任列表
firewall-cmd --zone=public --add-port=9090/tcp --permanent
// 重启防火墙
firewall-cmd --reload
// 进入文件夹下启动
./prometheus

浏览器访问 ip + 9090 可以出现页面，如下图所示。 ==若要想后台启动 Prometheus，则需要修改配置文件。具体操作可参考以下步骤。== 第一步、添加配置文件 在路径下 /usr/lib/systemd/system 创建 prometheus.service 文件，添加以下内容即可。

[Unit]
Description=https://prometheus.io

[Service]
Restart=on-failure
ExecStart=/opt/prometheus/prometheus --config.file=/opt/prometheus/prometheus.yml

[Install]
WantedBy=multi-user.target

第二步、刷新配置

// 重启防火墙
firewall-cmd --reload
// 重启服务配置文件
systemctl daemon-reload
// 启动prometheus
systemctl start prometheus
// 查看启动状态
ps -ef | grep prometheus

2、docker 安装 环境前置：已有docker环境 第一步、执行安装命令

docker run -d  -p 9090:9090 \
    -v /opt/prometheus/prometheus.yml:/etc/prometheus.yml \
    prom/prometheus
// 查看运行
docker ps -a

启动成功，如下图所示。

监控本身

在二进制文件中自带 prometheus.yml 中有监控自身配置。对应的内容如下

scrape_configs:
  - job_name: 'prometheus'

    # metrics_path defaults to '/metrics'
    # scheme defaults to 'http'.

    static_configs:
    - targets: ['localhost:9090']

可以访问 ip + 9090 /metrics，来查看具体的指标，如下图所示。
